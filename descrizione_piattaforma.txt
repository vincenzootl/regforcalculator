# regforcalculator — Descrizione piattaforma
Aggiornato: 2026-06-15

## Stack tecnico
- HTML5 + Vanilla CSS + Vanilla JS (no framework)
- PDF.js 2.16.105 (lettura PDF client-side)
- Chart.js 4.4.1 (grafici confronto e accantonamento)
- Architettura 100% client-side — zero upload, zero backend

## Struttura file
- index.html        → landing page pubblica
- calcolatore.html  → app a step (8 step: Profilo → Documenti → Parametri → Calcolo → F24 → Confronto → Accantonamento → Esporta PDF)
- css/style.css     → stile unico (dark mode, isomorphic flat, animazioni)
- js/parser.js      → logica di parsing PDF/XML e prefill campi
- js/calcolo.js     → calcolo fiscale (imposta sost., INPS GS, F24 giugno/dicembre)
- js/ui.js          → render risultati, grafici Chart.js

## Campi precompilati automaticamente
Quando l'utente carica i documenti, il sistema tenta di precompilare:
1. i-fatt       → fatturato corrente (da Totale Annuo FIC)
2. i-bolli      → n. fatture con bollo (da FIC: fattureCon > nFatture)
3. i-coeff      → coefficiente redditività (da LM/ATECO RPF)
4. i-aliq       → aliquota imposta sostitutiva (se 15% precompila, se 5% mantiene il 15% di default per prevenire errori di fine quinquennio e mostra un warning cliccabile per applicare il 5%)
5. i-inps-ded   → INPS deducibili (da LM35 RPF o totale cod.0900 F24 di tutti i periodi)
6. i-acc-imp    → acconti imposta sost. (da LM45 RPF o cod.1790+1791 F24)
7. i-acc-inps   → acconti INPS versati (da cod.0900 F24 per l'anno corrente o stimati da ricevute F24 e RPF)
8. i-credito    → credito anno prec. (da RX31 o LM43/LM44 RPF)
9. i-mesi       → mesi al F24 di giugno (calcolato automaticamente dalla data corrente)

## Documenti supportati e parser
### F24 (type='f24', PDF)
- Ricevuta: regex "Importo versamento : E. X,XX" → totaleVersato
- F24 digitale con codici tributo: scansione di ogni pagina separatamente con deduplicazione basata sulla firma dei record di pagamento estratti. Questo previene il conteggio multiplo dovuto alle tre copie (banca/contribuente/destinatario) presenti nel PDF.
- Distinzione INPS: i tributi 0900 vengono suddivisi in INPS deducibili (somma di tutti i pagamenti effettuati nell'anno) e Acconti INPS versati (solo i pagamenti riferiti all'anno fiscale corrente).

### RPF / Modello Redditi (type='rpf'|'redditi', PDF)
- Scansione per pagine:
  - Anno d'imposta estratto dal testo globale.
  - Pagina 5 (LM Sezione III): Individuazione dei codici ATECO a 6 cifre e scansione posizionale dei totali di fine quadro (lordo LM34, INPS deducibili LM35, netto LM36, imposta LM39) che seguono l'ultimo codice ATECO.
  - Coefficiente di redditività: estratto dal valore di 2 cifre che segue immediatamente i codici ATECO.
  - Aliquota imposta: calcolata direttamente dal rapporto matematico tra imposta LM39 e reddito netto LM36 (5% o 15%).
  - Credito residuo: estratto da quadro RX31 (pag.16) o fallback LM43/LM44 (pag.6).
  - Quadro RR: imponibile, INPS dovuto e saldo INPS estratti tramite l'ordinamento decrescente dei valori (Imponibile > Dovuto > Acconti > Saldo).

### FattureInCloud XML (type='fic', XML — formato SpreadsheetML)
Il file è un Excel-XML (SpreadsheetML) con namespace urn:schemas-microsoft-com:office:spreadsheet.
- Struttura: <Row> con <Cell><Data ss:Type="Number|String|DateTime">valore</Data></Cell>
- Strategia 1: individua la riga "Totale Annuo" → prima cella Number >= 100 = fatturato lordo annuo
- Strategia 2 (fallback): somma la prima cella numerica di ogni riga "Totale" mensile nelle sezioni "Fatture emesse"
- Conteggio fatture: conta ss:Type="DateTime" nelle sezioni "Fatture emesse" (esclude "Fatture ricevute")
- Bolli: per ogni Row con DateTime, controlla primo Number > 77,47 → conta come fattura con bollo
  Se impossibile discriminare, usa il totale fatture come fallback
- NOTA: da FIC si estraggono SOLO fatturato e n. fatture.
  INPS deducibili, acconti imp.sost. e credito provengono dai PDF RPF/F24.

## Merge intelligente
- Campi accumulabili (multi-F24): totaleVersato, acc0900, acc1790, acc1791, acc1792, fatt, nFatture, fattureCon
- Campi scalari (dichiarazione): assegnati una sola volta, non sovrascritti se già valorizzati
- Flag booleani (isRicevuta, isGrafico): OR tra file

## Logica prefill (priorità)
1. hasCodici (F24 digitale) → massima precisione per codice tributo
2. hasRicevuta / isGrafico (F24 ricevuta/grafico) → usa RPF + stima acconti INPS
3. Solo RPF → tutti i campi dalla dichiarazione

## Calcolo fiscale (calcolo.js)
- fattTot = fatt + nBolli × 2
- redLordo = fattTot × coeff/100
- redNetto = max(0, redLordo − inpsDed)
- imposta = max(0, redNetto × aliq/100)
- inpsDovCorrente = redLordo × inpsAliq/100
- inpsDovPrec = prevYear.inpsDov (da RR) se presente, altrimenti inpsDovCorrente
- saldoImp = max(0, imposta − accImp − credito)
- saldoInps = max(0, inpsDovCorrente − accInps)  ← usa INPS anno corrente, non prec.
- acc1Imp = floor(imposta × 50%), acc2Imp = round(imposta − acc1Imp)   [metodo storico]
- acc1Inps = floor(inpsDovCorrente × 40%), acc2Inps = floor(inpsDovCorrente × 40%)  [due rate uguali, totale 80% GS]
- F24 giugno = saldoImp + acc1Imp + saldoInps + acc1Inps
- F24 novembre = acc2Imp + acc2Inps

## Design
- Stile Isomorphic Flat (no shadow pesanti, no bordi doppi)
- Badge "da documento" / "manuale" su ogni campo precompilato
- Log real-time nell'area #extract-log con logOk/logWarn/logInfo/logErr
- Pills estratte in #extracted-pills (griglia)
- Tooltip modale per ogni campo (TIPS object)
- Campi auto-filled marcati con classe CSS .auto-filled

## Esportazione PDF / Stampa (Step 7)
- Ottimizzata via media query `@media print` in `css/style.css` per nascondere tutti gli elementi estranei (`.app-nav`, `.stepper`, `.btn-row`, etc.) e visualizzare solo `#step-7` e `#print-area`.
- Layout di stampa allineato A4 portrait con griglia KPI a 3 colonne fisse (`.print-grid-3`) e box F24 affiancati (`.print-f24-summary`).
- Funzione `exportPDF()` in `ui.js` che garantisce l'attivazione dello step 7 prima della stampa e applica un delay di 150ms per consentire il rendering finale.
